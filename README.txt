=======================
Case study - Smart depo
=======================

1. Jde o Server aplikaci, která je napsána pomocí Blazor Server,
	a. frontend je v 01_ServerBlazor,
	b. backend logika depa je v 10_SmartDepo.
2. Po spuštění lze nastavit výchozí počet tramvají s misí a bez mise.
3. Dále lze v aplikaci přidávat/odesílat tramvaje a plánovat jim mise.
4. Pro účely simulace jsou zde tlačítka s popiskem (5s), které simulují dlouhé vykonávání operace. Lze tedy otevřením aplikace ve více oknech testovat konkurenční volání akcí.
